# E-CITY 2050 교실 운영안

## 권장 환경

- 2차시, 차시당 45~50분
- 35명 학급, 4명 안팎 모둠
- 모둠별 스마트폰·태블릿·노트북 1대 이상
- 학교 Wi-Fi가 느리거나 일시적으로 끊길 수 있다고 가정
- 학생 개인정보는 입력하지 않고 모둠 이름만 사용

## 1차시: 발전 원리와 장소 적합성

| 시간 | 학생 활동 | 교사 개입 |
|---|---|---|
| 0~5분 | 첫 화면 핵심 질문 읽기 | “학교 콘센트 뒤에는 어떤 시설이 연결되어 있을까요?”라고 질문한다. |
| 5~15분 | 발전소에서 학교까지 전력 여정 확인, 2문항 응답 | 화력과 원자력은 열의 원인이 다르지만 증기·터빈·발전기를 이용한다는 공통점을 짚는다. |
| 15~30분 | 발전 기술 카드 4개 이상 확인 | “위험해서 안 된다” 하나로 끝내지 말고 공간·자연조건·안전시설·운송·비용을 함께 보게 한다. |
| 30~40분 | 학교 안·조건부 지역·외부 전력망으로 분류하고 이유 작성 | 원자력은 학교 안이 아니라 외부 전력망 공급원임을 확인한다. 수소는 생산 방식과 공급·안전 조건을 질문한다. |
| 40~50분 | 개념 확인 4문항과 모둠 설명 수정 | 정답만 읽지 않고 처음 생각과 달라진 문장을 한 가지 말하게 한다. |

## 2차시: 데이터 분석과 미래학교 설계

| 시간 | 학생 활동 | 교사 개입 |
|---|---|---|
| 0~8분 | 데이터 출처 상태 확인 후 수업용 데이터 또는 CSV 로드 | 실제 학교 데이터가 아니라면 반드시 수업용 가정 또는 지역 대체 자료로 표시하게 한다. |
| 8~18분 | 피크·최저 시간과 태양광 활동 시간 비교, 피크 원인 추론 | 그래프만으로 원인을 확정할 수 없으며 추가 자료가 필요할 수 있음을 말한다. |
| 18~33분 | 태양광·ESS·수소·절감·스마트 관리 조절, 설계 A/B 저장 | ESS는 발전량이 아니며 낮의 잉여를 저장해 이후 부족 시간에 사용한다는 흐름을 확인한다. |
| 33~40분 | 두 설계 비교 후 최종안 선택 | 지역 충당률만 높이는 경쟁이 되지 않게 외부망·장소 조건·한계를 함께 비교시킨다. |
| 40~50분 | 보고서 보완, 1분 발표 | 학교 안·조건부·외부 기술의 구분 이유와 실제 적용 전 추가 조사를 반드시 말하게 한다. |

## 속도 차이 대응

- 빠른 모둠: 수소 생산 방식을 `green`, `mixed`, `unspecified`로 바꿨을 때 설명이 어떻게 달라지는지 비교한다.
- 보통 모둠: 수업용 데이터로 A/B 설계와 보고서까지 완료한다.
- 도움이 필요한 모둠: 태양광·ESS·절감만 먼저 조절하고 외부 전력망 부족분을 읽게 한다.

## 기기·네트워크 비상 대안

- CSV가 열리지 않으면 `수업용 가정 데이터` 버튼으로 즉시 전환한다.
- 인터넷이 끊겨도 이미 열린 앱은 외부 API 없이 핵심 활동을 진행할 수 있다.
- 앱 접속 자체가 안 되면 수업용 24시간 표와 기술 분류표를 인쇄해 같은 순서로 활동한다.
- 한 모둠 기기가 멈추면 다른 모둠의 설계 화면을 보며 종이 활동지에 A/B 값을 기록한다.

## 수업 후 데이터 처리

- 학생 제출 대상: 모둠 보고서 복사본 또는 인쇄물
- 교사용 보관이 필요한 경우에만 교사 모드에서 JSON 파일을 저장
- 저장하지 않는 정보: 학생 이름, 학번, 연락처, 개인 식별 정보
- 공용 기기에서는 수업 종료 후 `처음부터`를 실행하고 확인한 뒤 탭을 닫는다.
- 학생 진행은 현재 탭의 sessionStorage에 자동 저장된다. 지역 공공데이터 원본은 교사가 별도로 관리한다.

## 수업 전 수동 확인

- 학생 URL에는 `?mode=teacher`가 붙지 않았는지 확인
- 수업용 데이터 배지와 900 kWh 합계 확인
- 실제 CSV 사용 시 제공기관·자료명·기준일·지역 단위 입력
- 모바일 화면에서 가로 스크롤과 긴 파일명 표시 확인
- 인쇄 미리보기에서 보고서 핵심 항목이 한 흐름으로 출력되는지 확인
