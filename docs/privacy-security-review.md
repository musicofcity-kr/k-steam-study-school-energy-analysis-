# 학생 사용 개인정보·보안 검토

## 판정

학생 기본 URL은 개인정보를 입력하지 않는 모둠 활동에 사용할 수 있다. 교사용 모드는 인증 화면이 아니므로 민감정보나 평가 정답을 저장하지 않는다.

## 저장하는 데이터

| 데이터 | 저장 위치 | 운영 기준 |
|---|---|---|
| 미션 답, 에너지 데이터, 설계안, 보고서 | 현재 탭 sessionStorage | 수업 종료 후 `처음부터` 실행 및 탭 닫기 |
| 교사가 명시적으로 저장한 상태 | localStorage | 교사 모드에서만 사용하고 공용기기에서는 삭제 |
| 학생 개인정보 | 저장하지 않음 | 실명·학번·연락처 대신 모둠 별칭 사용 |

## 외부 전송

- 서버 제출 API, 분석 SDK, 로그인 DB와 외부 API 호출이 없다.
- CSV는 브라우저 안에서만 읽으며 1MB 이하로 제한한다.
- 외부 웹폰트를 사용하지 않아 앱을 여는 것만으로 제3자 CDN에 요청하지 않는다.

## 권한과 무결성

- 학생은 쿼리가 없는 기본 URL을 사용한다.
- `?mode=teacher`는 로컬 가정값 조절용이며 인증 또는 권한 경계가 아니다.
- 학생 모드는 저장된 교사용 가정값을 무시하고 기본 수업 가정값을 사용한다.
- 중앙 성적 산출이나 고부담 평가에는 이 앱의 로컬 결과만 사용하지 않는다.

## 수업 전 확인

- 학생 URL에 `?mode=teacher`가 없는지 확인한다.
- 모둠 별칭 사용과 개인정보 입력 금지를 안내한다.
- 공용기기에서 이전 모둠 기록이 보이면 `처음부터`를 실행한다.
- 공개 배포 URL의 HTTPS와 보안 응답 헤더는 실제 배포 후 별도로 확인한다.
