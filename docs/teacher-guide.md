# 교사용 안내서

## 수업 목적

학생이 전력 사용 데이터를 보고 피크 시간과 사용 패턴을 찾은 뒤, 에너지원의 역할과 한계를 비교하여 2050년 탄소중립 미래 학교구역 설계안을 설명하도록 돕습니다.

이 앱은 실제 도시 에너지 설계 도구가 아니라 중1 수업용 비교 도구입니다.

## 수업 전 준비

1. 앱 실행 환경을 준비합니다.

```bash
npm install
npm run dev
npm test
npm run build
```

2. 학생에게 제공할 데이터 방식을 정합니다.
   - 빠른 체험: `sample_energy_usage_practice.csv`
   - 실제 자료 활동: 교사가 내려받고 정리한 공공데이터 CSV

3. 샘플 데이터 안내를 먼저 확인합니다.
   - 샘플 CSV는 수업용 가정 데이터입니다.
   - 실제 공공데이터가 아닙니다.
   - 실제 지역이나 학교의 전력 사용량으로 해석하지 않게 지도합니다.

## 공공데이터 준비 흐름

1. 서울 열린데이터광장 또는 공공데이터포털에서 전력 사용량 관련 CSV를 내려받습니다.
2. 수업에 필요한 열만 남깁니다.
3. 날짜, 시간, 지역, 전력사용량 열을 확인합니다.
4. 앱에 업로드합니다.
5. 앱이 열을 자동으로 찾지 못하면 컬럼 매핑에서 직접 지정합니다.
6. 학생 보고서의 데이터 출처 칸에 자료 출처를 적게 합니다.

개원중학교 데이터 수집팩을 사용할 때는 앱의 `개원중 업로드 템플릿`을 내려받아 사용합니다. 이 파일은 입력 틀일 뿐이며 `usage_kWh` 값이 비어 있습니다. 데이터팩에서 지정한 원천자료는 `서울특별시 법정동별시간별전력사용량`이고, 출처 URL은 `https://data.seoul.go.kr/dataList/OA-22835/F/1/datasetView.do`입니다. 필터는 `법정동/동명=개포동`입니다. 해당 행을 추출한 뒤 표준 출력 컬럼 `date,hour,region,usage_kWh` 형식으로 실제 값을 채워 넣어야 합니다.

MVP 수업에서는 시간별 전력 사용량 CSV를 우선 사용합니다. 태양광 발전소 현황, 전기차 충전소 현황, 수소충전소 현황, 일사량 또는 일조시간 CSV는 확장 활동으로 다루는 것이 적절합니다.

## 권장 CSV 열

```csv
date,hour,region,usage_kWh
```

또는

```csv
날짜,시간,지역,전력사용량_kWh
```

필수에 가까운 열은 시간과 전력사용량입니다. 날짜와 지역은 보고서 맥락을 돕는 보조 정보입니다.

## 교사용 가정값 설명

교사용 설정 패널의 값은 공식값이 아니라 수업용 비교 가정값입니다.

- 태양광 100% 설치 시 시간당 평균 공급량: 학교 옥상, 체육관, 주차장 지붕 등을 모두 활용했다고 보는 비교값
- 수소 100% 활용 시 시간당 평균 공급량: 장시간 안정적 전력 공급을 토론하기 위한 비교값
- 차세대 원자력 100% 활용 시 시간당 평균 공급량: 미래 기술 선택지를 균형 있게 토론하기 위한 가상 비교값
- 에너지 절감 최대 비율: LED 조명, 단열, 냉난방 효율, 대기전력 줄이기 등을 반영한 수업용 상한
- 탄소배출계수: 절감 효과를 비교하기 위한 입력값

지도 문구:

> 이 값은 수업용 비교를 위한 가정값입니다. 실제 도시 설계에는 지역 일사량, 설치 면적, 설비 효율, 법과 안전 기준, 비용 분석 등이 추가로 필요합니다.

ESS는 발전원이 아니므로 총 발전량에 단순 합산하지 않습니다. 학생에게 ESS는 "전기를 만들어 내는 장치"가 아니라 "남는 전기를 저장했다가 필요한 시간에 쓰도록 돕는 장치"라고 안내합니다.

공급량은 시간당 평균 공급량에 업로드된 유효 행 수를 곱해 계산합니다. 태양광도 수업용 단순화를 위해 시간당 평균 공급으로 처리하며, 실제 태양광은 낮과 날씨, 계절에 따라 달라진다고 안내합니다.

수소와 차세대 원자력 공급은 수업 단순화를 위해 무배출로 가정합니다. 실제 수소는 생산 방식(그레이/블루/그린)에 따라 배출이 다르므로 심화 토론에서 구분합니다.

## 수업 운영 흐름

1. 시작 화면에서 오늘의 미션을 확인합니다.
2. 예시 데이터 또는 교사가 준비한 CSV를 불러옵니다.
3. 그래프와 요약 지표에서 피크 시간과 사용 패턴을 찾습니다.
4. 에너지원 카드로 장점, 한계, 도시 설계에서의 역할을 비교합니다.
5. 설계실에서 슬라이더를 조절하며 자립률과 점수 변화를 관찰합니다.
6. 결과 보고서를 복사하거나 인쇄해 활동지와 발표에 사용합니다.

## 토의 질문 예시

- 전기를 가장 많이 쓰는 시간은 언제인가?
- 그 시간에 전력 사용량이 높은 이유는 무엇일까?
- 태양광은 어떤 시간대에 도움이 될까?
- ESS는 왜 발전량에 더하지 않을까?
- 에너지 절감 전략은 왜 도시 설계에서 중요할까?
- 우리 설계의 장점과 한계는 무엇인가?

## 보고서 지도

학생 보고서에는 데이터 출처를 반드시 구분하게 합니다.

- 샘플 사용 시: "수업용 가정 데이터, 실제 공공데이터 아님"
- 개원중학교 팩 템플릿 사용 시: "개포동 시간별 전력사용량을 학교 주변 대체 지표로 사용, 학교 자체 전력량 아님"
- 실제 CSV 사용 시: 기관명, 자료명, 내려받은 날짜 또는 파일명

핵심 전략 3가지는 에너지원 이름만 쓰지 않고 "왜 선택했는지"까지 적게 합니다.

예:

- 피크 시간이 오후 2시라서 낮 시간 태양광 활용을 늘렸다.
- 밤과 흐린 날을 대비해 ESS (전기 저장소)로 피크 대응 점수를 높였다.
- 전력 사용 자체를 줄이기 위해 LED와 단열 개선을 선택했다.

## 수업 적용 전 체크리스트

- [ ] 샘플 CSV가 실제 공공데이터가 아님을 안내했다.
- [ ] 실제 자료를 사용할 경우 출처를 확인했다.
- [ ] 시간 열과 전력사용량 열이 앱에서 인식되는지 확인했다.
- [ ] 교사용 가정값을 공식값처럼 설명하지 않았다.
- [ ] ESS를 발전량으로 합산하지 않는다는 점을 설명했다.
- [ ] 결과 점수가 수업용 비교 지표임을 안내했다.
- [ ] 학생 개인정보가 CSV와 보고서에 들어가지 않도록 했다.

## 한계

이 앱은 중1 수업에서 데이터 기반 의사결정을 경험하게 하는 도구입니다. 실제 도시 에너지 설계에는 지역 일사량, 건물 면적, 설비 효율, 안전 기준, 전력망 조건, 경제성 분석, 정책 검토가 필요합니다.

차세대 원자력은 무조건 좋은 해결책으로 제시하지 않습니다. 역할, 장점, 한계, 안전성 검토 필요성을 함께 비교하는 토론 항목으로 다룹니다.
