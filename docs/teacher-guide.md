# 교사용 안내서

## 수업 목적

학생이 발전 원리와 입지 조건을 이해하고 발전·저장·절감·관리·외부 공급을 구분한 뒤, 전력 사용 데이터와 장소 적합성을 근거로 미래학교 에너지 시스템을 설명하도록 돕습니다.

## 실행과 모드

```bash
npm install
npm run dev
npm test
npm run build
```

- 학생 모드: 기본 URL
- 교사용 가정값 설정: URL 끝에 `?mode=teacher`
- 학생에게는 교사용 쿼리가 없는 URL을 제공합니다.
- 교사용 쿼리는 인증 기능이 아닙니다. 민감정보나 평가 정답을 두지 말고, 학생 모드는 저장된 교사용 가정값을 사용하지 않습니다.

## 데이터 준비

- 빠른 체험: `public/sample-data/sample_energy_usage_practice.csv`
- 공공데이터: 교사가 내려받고 `date,hour,region,usage_kWh`로 정리한 CSV
- 개원중 템플릿: 값이 비어 있는 입력 틀이며 실제 학교 전력량이 아님

공공데이터를 선택하면 제공기관, 자료명, 기준일, 지역 단위, 학교 자체/지역 대체 여부를 입력합니다. 파일명만으로 공공데이터라고 판정하지 않습니다.

개포동 자료를 사용할 때는 `서울특별시 법정동별시간별전력사용량`에서 `법정동/동명=개포동`을 추출하며, 학교 자체 전력량이 아닌 지역 대체 자료라고 표시합니다.

## 교사용 가정값

모든 값은 공식값이 아니라 수업용 비교 가정값입니다.

- 태양광 100% 설치 시 활동 시간당 공급량
- 조건부 수소 연료전지의 시간당 공급량
- ESS 최대 용량과 시간당 충전·방전 한계
- ESS 왕복 효율
- 절감 최대 비율
- 스마트 관리의 피크 추가 절감 상한
- 태양광 발전 가능 시작·끝 시간
- 수소 생산 방식: green, mixed, unspecified

수소 생산 방식이 확인되지 않으면 환경성 점수를 가산하지 않습니다. 실제 수소 환경성은 생산 전력과 원료, 운송을 함께 봐야 합니다.

## 지도 핵심

- 화력과 원자력은 열의 원인이 다르지만 증기·터빈·발전기를 이용한다.
- 태양광은 터빈 없이 빛을 직접 전기로 바꾼다.
- 대규모 발전시설 입지는 위험 하나가 아니라 공간·자연조건·안전시설·운송·비용을 함께 본다.
- 원자력은 학교 안 설비가 아니라 외부 전력망 공급원의 한 종류다.
- 수소 연료전지는 일반 학교에 바로 적용하는 기술이 아니라 조건부 지역 보조 기술이다.
- ESS 충전·방전은 발전량이 아니다.
- 외부 전력망은 현실적인 연결 시스템이며 실패 표시가 아니다.

## 계산 해석

태양광은 설정한 활동 시간에만 발전합니다. 시간별 계산은 절감·스마트 관리, 태양광·수소 직접 사용, 태양광 잉여 ESS 충전, ESS 방전, 외부 전력망, 잉여 전력 순서입니다.

```text
지역 에너지 충당률 =
(절감 후 소비량 - 외부 전력망 사용량)
/ 절감 후 소비량 * 100
```

지역 충당률에 학교 밖 원자력이나 외부 전력망 공급을 포함하지 않습니다.

## 수업 운영

권장 2차시 운영안과 기기 오류 대안은 `docs/classroom-operation.md`를 사용합니다.

1차시는 발전 원리와 장소 분류, 2차시는 데이터 분석·A/B 설계·보고서에 배정합니다. 속도가 느린 모둠은 태양광·ESS·절감 세 항목으로 먼저 시스템 흐름을 완성하게 합니다.

## 수업 적용 전 체크리스트

- [ ] 학생 URL에 `?mode=teacher`가 없다.
- [ ] 수업용 데이터가 실제 학교 전력량이 아님을 안내했다.
- [ ] 실제 CSV 제공기관과 자료명을 확인했다.
- [ ] 원자력을 학교 안 시설로 표현하지 않았다.
- [ ] ESS와 스마트 관리를 발전량으로 설명하지 않았다.
- [ ] 수소 생산 방식이 미확인일 때 환경성을 단정하지 않았다.
- [ ] 보고서에 장소 구분 이유와 설계 한계가 포함됐다.
- [ ] 공용 기기는 수업 후 `처음부터`를 실행하고 탭을 닫는다.
- [ ] 학생 개인정보가 CSV와 보고서에 들어가지 않았다.
- [ ] 학생은 실명·학번이 아닌 모둠 별칭을 사용한다.
