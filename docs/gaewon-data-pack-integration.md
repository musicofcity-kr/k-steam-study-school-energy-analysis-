# 개원중학교 데이터 수집팩 반영 기록

## 반영 원칙

데이터 팩은 실제 전력사용량 완성 데이터가 아니라, 개원중학교 기준의 수집·전처리 틀이다. 따라서 앱의 그래프와 자립률 계산에 빈 값이나 학교 프로필을 억지로 넣지 않았다.

## 앱 보조자료로 반영한 항목

계산·그래프용 데이터로 직접 반영한 파일은 없다. 아래 항목은 앱의 안내, 교사용 다운로드, 학교 컨텍스트 표시를 위한 보조자료다.

- `public/sample-data/gaewon_energy_usage_upload_template.csv`
  - 용도: 교사용 전력사용량 업로드 템플릿
  - 이유: 앱의 표준 구조 `date,hour,region,usage_kWh`와 맞다.
  - 주의: `usage_kWh`가 비어 있으므로 그 자체로 그래프 데이터가 아니다.
- `src/data/gaewonSchoolContext.ts`
  - 용도: 시작 화면의 대상 학교, 지역 필터, 한계 문구 표시
  - 반영 값: 개원중학교, 강남구 개포동, 학교코드 7091420, 학생수 966명, 교원수 65명
  - 전력 원천자료: `서울특별시 법정동별시간별전력사용량`, 필터 `법정동/동명=개포동`, 표준 컬럼 `date,hour,region,usage_kWh`
  - 출처 URL: `https://data.seoul.go.kr/dataList/OA-22835/F/1/datasetView.do`
  - 주의: 학생수와 교원수는 공시 시기별로 변할 수 있다.

## 문서/참고로만 사용한 항목

- `gaewon_school_profile.csv`
  - 학교명, 주소, 학교코드, 법정동 필터 등 일부만 앱 컨텍스트에 반영했다.
  - 전화번호, 팩스 등 앱 활동에 직접 필요 없는 값은 제외했다.
- `gaewon_teacher_assumptions.csv`
  - 학교명, 지역 필터, 대체 지표 한계만 반영했다.
  - 실제 학교 일일 전력사용량, 태양광 면적, ESS 용량은 빈 값이므로 제외했다.
- `gaewon_local_filter.csv`
  - 1순위 전력 원천자료명과 필터 조건은 앱 메타데이터로 반영했다.
  - 실제 전력값은 포함되어 있지 않아 계산 데이터에는 넣지 않았다.
- `gaewon_public_data_collection_plan.csv`
  - 1순위 수집 계획은 앱 메타데이터와 교사용 안내에 반영했다.
  - 나머지 태양광·기상·충전소 자료는 후속 확장 후보로 남겼다.
- `gaewon_student_dataset_map.csv`
- `README_개원중학교_데이터수집가이드.md`
- `E-CITY2050_개원중학교_데이터수집팩.xlsx`
  - 위 CSV와 가이드를 묶은 교사용 원본 팩으로 보고, 앱 런타임 데이터에는 넣지 않았다.

## 제외한 항목과 이유

- 빈 `usage_kWh` 값
  - 이유: 실제 전력 사용량처럼 보일 위험이 있고, 총사용량·피크 시간·자립률을 왜곡한다.
- 실제 학교 전력사용량 관련 빈 항목
  - 이유: 공개 공공데이터만으로 즉시 확인되지 않은 값이다.
- 태양광 지붕 면적, 태양광 용량, ESS 용량 빈 항목
  - 이유: 공식값처럼 보이면 안 되는 교사용 심화 입력값이다.
- 전화번호, 팩스
  - 이유: 학생 활동과 계산 모델에 필요하지 않다.

## 수업 적용 안내

교사는 서울특별시 법정동별시간별전력사용량 원본에서 `개포동` 행만 추출하고, 템플릿의 `usage_kWh` 값을 채운 뒤 앱에 업로드한다. 학생 보고서에는 반드시 “학교 자체 전력량이 아니라 학교 주변 법정동 자료를 사용했다”고 적게 한다.
