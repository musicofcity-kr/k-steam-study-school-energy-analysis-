# E-CITY 2050 미래학교 에너지 시스템

[![CI](https://github.com/musicofcity-kr/k-steam-study-school-energy-analysis-/actions/workflows/ci.yml/badge.svg)](https://github.com/musicofcity-kr/k-steam-study-school-energy-analysis-/actions/workflows/ci.yml)

중학교 1학년 학생이 현재의 발전방식과 전력 전달 과정을 이해하고, 발전·저장·절감·관리·외부 공급을 구분한 뒤, 시간별 전력 데이터를 근거로 2050 미래학교 에너지 시스템을 설계하는 React/Vite/TypeScript 수업용 웹앱입니다.

이 도구는 실제 학교·도시 설계를 대체하는 공학 계산 도구가 아닙니다. 모든 기본값과 점수는 수업용 비교 가정입니다.

## 실행

```bash
npm install
npm run dev
npm test
npm run build
```

학생 화면은 기본 URL을 사용합니다. 교사용 가정값은 URL에 `?mode=teacher`를 붙였을 때만 표시됩니다.

## 수업 흐름

1. `전기는 어디서?`: 발전소→송전선→변전소→지역 전력망→학교와 발전 원리 확인
2. `학교 주변 적합성`: 학생은 태양광·풍력·수소 연료전지·원자력 필수 4개를 학교 안·조건부 지역·외부 전력망으로 분류
3. `전력 패턴 탐정`: 수업용 데이터 또는 CSV를 불러와 피크·최저 시간과 피크 원인 추론
4. `미래학교 시스템`: 태양광·ESS·조건부 수소·절감·스마트 관리를 조절하고 설계안 A/B 비교
5. `보고서 & 발표`: 데이터 출처, 장소 적합성, 외부 공급, 장점·한계·추가 조사를 포함한 결과물 작성

## 핵심 구분

| 역할 | 기술 |
|---|---|
| 발전 | 화력, 원자력, 수력, 풍력, 태양광, 수소 연료전지 |
| 저장 | ESS |
| 절감 | LED, 단열, 냉난방 효율, 대기전력 차단 |
| 관리 | 스마트 에너지 관리 |
| 공급·연결 | 외부 전력망, 마이크로그리드 개념 |

원자력은 학교 안 설계 슬라이더에 포함하지 않습니다. 학교 밖 발전소에서 생산되어 전력망을 통해 공급되는 전원으로만 다룹니다. 수소 연료전지는 공급·안전·공간·비용·생산 방식이 확인된 조건부 지역 보조 기술입니다.

## CSV와 데이터 출처

권장 형식:

```csv
date,hour,region,usage_kWh
2026-07-08,14,개포동,305
```

- 학생 모드: `수업 데이터로 시작하기` 또는 `선생님이 준 CSV 열기`
- 교사 모드: 한국어 열 이름과 수동 컬럼 매핑, 출처 상세 입력 지원
- 1MB를 넘는 CSV 거부, 잘못된 파일 선택 시 기존 정상 데이터 유지
- UTF-8, UTF-8 BOM, CP949/EUC-KR 디코딩 지원
- 일부 잘못된 행 제외 및 제외 행 수 안내
- 출처 상태: `수업용 가정 데이터`, `교사가 준비한 공공데이터`, `출처 확인 필요`
- 공공데이터 상세: 제공기관, 자료명, 기준일, 지역 단위, 학교 자체/지역 대체 여부

`public/sample-data/sample_energy_usage_practice.csv`는 총 900 kWh, 피크 14시 67 kWh인 24시간 수업용 가정 데이터입니다. 실제 학교나 지역 전력량이 아닙니다.

개원중학교 데이터 수집팩에는 실제 전력량이 없으므로 앱에는 빈 업로드 템플릿과 출처 안내만 포함합니다. `서울특별시 법정동별시간별전력사용량`에서 `법정동/동명=개포동` 자료를 사용할 경우에도 학교 자체 전력량이 아닌 지역 대체 자료라고 표시해야 합니다.

## 시간별 에너지 모델

각 행에서 절감→스마트 피크 관리→낮 시간 태양광→조건부 수소→직접 사용→태양광 잉여 ESS 충전→ESS 방전→외부 전력망→잉여 전력 순서로 계산합니다.

- 태양광은 설정된 활동 시간에만 발전
- ESS는 용량·시간당 충전·방전 한계·왕복 효율 적용
- ESS 방전량은 발전량으로 합산하지 않음
- 스마트 관리는 발전량이 아니라 피크 수요 조절에 반영
- 수소는 생산 방식이 `green`일 때만 환경성 비교 점수에 제한적으로 반영

```text
지역 에너지 충당률 =
(절감 후 소비량 - 외부 전력망 사용량)
/ 절감 후 소비량 × 100
```

외부 전력망 사용량은 지역 충당률에 포함하지 않으며, 전력망 사용을 실패한 설계로 평가하지 않습니다.

## 설계와 결과물

- 설계안 A/B 저장·불러오기·최종 선택
- 지역 에너지 충당률, 외부 전력망 사용량, 피크 대응 정도 우선 표시
- 태양광 생산, ESS 충전·방전, 잉여 전력과 밤 시간 외부망 의존 상세
- 활동지 값 복사
- 학생 보고서 자동 초안, 직접 수정, 복사와 인쇄
- 교사 모드 JSON 저장과 명시적 브라우저 저장
- 수동 수정 뒤 현재 데이터·설계를 반영하는 자동 초안 재생성
- 선행 미션 완료 전 다음 미션과 A/B 설계 조작 잠금
- 연습 데이터의 `수업용 가정 데이터` 출처 표시 고정
- 수업용 데이터 보고서 상단 자동 경고
- 현재 팀 설계 초기화와 전체 수업 초기화
- 학생 진행은 현재 탭의 sessionStorage에 자동 저장하고 새로고침 후 이어서 진행
- 교사 모드의 명시적 저장은 localStorage `version: 2`와 구버전 상태 마이그레이션 지원

## 개인정보와 보안

- 서버 저장과 학생 개인정보 수집을 하지 않습니다.
- 학생 이름·학번·연락처를 CSV나 보고서에 넣지 않고 모둠 별칭만 사용합니다.
- 공용 기기에서는 수업 후 `처음부터` 확인 절차로 sessionStorage와 localStorage를 비웁니다.
- 외부 API·분석 SDK·외부 웹폰트 요청이 없습니다.
- API 키를 사용하지 않습니다.

## 한계

- 실제 발전량은 날씨, 계절, 설치 방향과 면적에 따라 달라집니다.
- 실제 ESS 운전에는 배터리 종류, 열관리, 안전, 수명과 제어 전략이 필요합니다.
- 실제 수소 환경성은 생산 원료·전력, 저장과 운송을 함께 평가해야 합니다.
- 실제 설계에는 비용, 법, 안전, 전력망, 지역 수용성과 전문 검토가 필요합니다.
- 수업 적용 전 교사가 데이터 출처와 가정값을 확인해야 합니다.
