# E-CITY 2050 Energy Design Lab

[![CI](https://github.com/musicofcity-kr/k-steam-study-school-energy-analysis-/actions/workflows/ci.yml/badge.svg)](https://github.com/musicofcity-kr/k-steam-study-school-energy-analysis-/actions/workflows/ci.yml)

중학교 1학년 학생이 전력 사용 CSV 데이터를 읽고, 에너지원의 장단점을 비교한 뒤, 2050년 탄소중립 미래 학교구역 설계안을 설명해 보는 수업용 STEAM 웹앱입니다.

## 상태

- 문서 상태: prototype
- 대상 수업: 중1 융합수업, 에너지 데이터 탐색, 탄소중립 미래도시 설계
- 배포 형태: Vite React TypeScript 정적 프론트엔드 앱

이 도구는 실제 도시 에너지 설계를 대체하는 전문 공학 계산 도구가 아닙니다. 중학교 1학년 수업에서 데이터를 읽고 비교하며 근거 있는 의사결정을 연습하기 위한 교육용 비교 도구입니다.

## 샘플 데이터 주의

`public/sample-data/sample_energy_usage_practice.csv`는 수업용 가정 데이터입니다. 하루 총 900 kWh, 피크 14시 67 kWh인 중학교 1일 전력 사용 패턴으로 수업 PPT와 활동지 숫자에 맞춥니다.

- 실제 공공데이터가 아닙니다.
- 실제 지역, 학교, 도시의 전력 사용량을 나타내지 않습니다.
- 수업 전 교사가 서울 열린데이터광장, 공공데이터포털, 기상자료개방포털 등에서 내려받은 CSV로 교체할 수 있습니다.
- 화면이나 보고서에서 샘플 데이터를 사용할 때는 데이터 출처를 "수업용 가정 데이터"로 적어야 합니다.

```bash
npm install

npm test

```

- `npm install`: 의존성 설치
- `npm run dev`: 개발 서버 실행
- `npm test`: CSV 파싱, 에너지 모델, 보고서 생성 단위 테스트 실행
- `npm run build`: TypeScript 검사와 정적 빌드 실행

## 주요 기능

- 연습 데이터 불러오기
- 교사가 준비한 CSV 업로드
- 개원중학교 데이터 수집팩 기반 업로드 템플릿 제공
- 날짜, 시간, 지역, 전력사용량 열 자동 감지 및 수동 매핑

- 태양광, ESS (전기 저장소), 수소, 차세대 원자력, 절감률 조절을 통한 교육용 시나리오 비교
- 외부 전력망 부족분과 활동지에 적을 값 요약
- 학생 보고서 초안 생성, 복사, 인쇄, JSON 저장
- 교사용 가정값 설정

## CSV 형식

우선 지원하는 핵심 데이터는 시간별 전력 사용량 CSV입니다.

권장 열 이름:

```csv
date,hour,region,usage_kWh
수업용가정,14,일반 중학교(수업용 가정),67
```

한국어 열 이름도 사용할 수 있습니다.

```csv
날짜,시간,지역,전력사용량_kWh
2026-07-08,14,우리지역,305
```

열 이름이 다르면 앱에서 시간 열과 전력사용량 열을 직접 지정합니다.

## 개원중학교 데이터 수집팩 반영

`E-CITY2050_개원중학교_데이터수집팩.zip`은 실제 전력사용량 데이터가 아니라 개원중학교 기준의 수집·전처리 틀입니다. 그래서 계산·그래프용 데이터로 직접 넣은 파일은 없고, 앱 보조자료로만 다음을 반영했습니다.

- 시작 화면에 개원중학교, 강남구 개포동 기준과 한계 문구 표시
- `public/sample-data/gaewon_energy_usage_upload_template.csv` 교사용 다운로드 제공
- 업로드 화면에 팩에서 찾은 원천자료 `서울특별시 법정동별시간별전력사용량`, 필터 `법정동/동명=개포동`, 표준 컬럼 `date,hour,region,usage_kWh` 표시
- 원천자료 URL: `https://data.seoul.go.kr/dataList/OA-22835/F/1/datasetView.do`
- `docs/gaewon-data-pack-integration.md`에 포함/참고/제외 판단 기록

템플릿은 `usage_kWh` 값이 비어 있으므로 그 자체로 그래프 데이터가 아닙니다. 팩에는 빈 업로드 템플릿만 있고 실제 학교 전력값은 포함되어 있지 않습니다. 교사가 실제 개포동 시간별 전력사용량 값을 채운 뒤 업로드해야 합니다.

## 교사용 데이터 준비 흐름

1. 서울 열린데이터광장의 `서울특별시 법정동별시간별전력사용량` 자료를 확인합니다. URL: `https://data.seoul.go.kr/dataList/OA-22835/F/1/datasetView.do`
2. 법정동/동명 필터로 `개포동` 행만 남깁니다.
3. 날짜, 시간, 지역, 전력사용량 열을 `date,hour,region,usage_kWh`로 맞춥니다.
4. 앱에 CSV를 업로드합니다.
5. 컬럼 매핑이 필요하면 앱에서 직접 지정합니다.
6. 학생 보고서에는 실제 데이터 출처 또는 "수업용 가정 데이터"를 구분해 적게 합니다.

## 교사용 가정값

교사용 설정값은 수업용 비교를 위한 가정값입니다. 실제 공식값처럼 사용하면 안 됩니다.

- 태양광 100% 설치 시 시간당 평균 공급량
- 수소 100% 활용 시 시간당 평균 공급량
- 차세대 원자력 100% 활용 시 시간당 평균 공급량
- 에너지 절감 최대 비율
- 탄소배출계수 입력값
- 태양광 발전 가능 시간대 표시 시작/끝 시간

공급량은 `시간당 평균 공급량 × 업로드된 유효 행 수 × 슬라이더 비율`로 계산합니다. 태양광도 수업용 단순화를 위해 시간당 평균 공급으로 가정합니다. 실제 태양광은 낮과 날씨, 계절에 따라 달라집니다.
그래프의 "태양광이 일하는 시간" 음영은 밤에는 ESS와 다른 에너지가 필요하다는 오개념 방지용 시각 보조입니다. 공급량 계산은 시간대와 무관한 수업용 단순화 그대로 유지합니다.

ESS (전기 저장소)는 발전량에 직접 더하지 않습니다. 이 앱에서는 ESS를 전기를 저장해 피크 시간 대응을 돕는 장치로 다룹니다.

## 학생 활동 흐름

1. 공공데이터 또는 수업용 가정 데이터를 불러옵니다.
2. 언제 전기를 가장 많이 쓰는지 전력 사용 패턴을 찾습니다.
3. 여러 에너지원의 장점과 한계를 비교합니다.
4. 미래 학교구역의 에너지 조합을 설계합니다.
5. 데이터 근거를 바탕으로 보고서와 1분 발표문을 작성합니다.

## 개인정보와 보안

- MVP는 서버 저장을 사용하지 않습니다.
- 학생 개인정보를 CSV나 보고서에 넣지 않습니다.
- 브라우저 localStorage 또는 JSON 파일 저장은 같은 기기에서 다시 확인하기 위한 수업 편의 기능으로만 사용합니다.

## 한계

- 이 앱의 자립률, 피크 대응 점수, 다양성, 환경성, 현실성 점수는 수업용 비교 지표입니다.
- 실제 도시 설계에는 지역 일사량, 설치 면적, 설비 효율, 법과 안전 기준, 비용 분석, 전력망 조건이 추가로 필요합니다.
- 수소·차세대 원자력 공급은 수업 단순화를 위해 무배출로 가정합니다. 실제 수소는 생산 방식(그레이/블루/그린)에 따라 배출이 다릅니다.
- 차세대 원자력은 설치 권장이 아니라 미래 기술 선택지로 토론하기 위한 항목입니다.
- 점수 계산식과 계수 한계는 `docs/score-model.md`에 정리했습니다.
- MVP의 우선순위는 시간별 전력 사용량 CSV이며, 태양광 발전소 현황, 충전소 현황, 기상 데이터 등은 후속 확장 대상입니다.
- 보고서 PNG 저장은 MVP에 포함하지 않았고, JSON 저장과 인쇄를 우선 제공합니다.
