# E-CITY 2050 미래학교 에너지 시스템
## School Energy Data Analysis & Future System Design Lab

[![CI](https://github.com/musicofcity-kr/k-steam-study-school-energy-analysis-/actions/workflows/ci.yml/badge.svg)](https://github.com/musicofcity-kr/k-steam-study-school-energy-analysis-/actions/workflows/ci.yml)

> **중학교 1학년 학생이 전기의 생산과 전달 과정을 이해하고, 시간별 전력 사용 데이터를 분석한 뒤, 발전·저장·절감·관리·외부 공급을 연결하여 2050 미래학교의 에너지 시스템을 설계하는 K-STEAM 수업용 웹 애플리케이션입니다.**

- 대상: 중학교 1학년 중심의 과학·기술·수학·정보·환경 융합수업
- 권장 수업: 2차시, 모둠 활동
- 기본 흐름: **개념 이해 → 장소 적합성 판단 → 데이터 분석 → A/B 설계 → 보고서·발표**
- 주요 기술: React · TypeScript · Vite · Papa Parse · Recharts
- 데이터 처리: 브라우저 내부 처리
- 서버·로그인·API 키: 사용하지 않음
- 개발 단계: 교실 적용 전 검증이 필요한 수업용 MVP

> 이 프로그램은 실제 학교나 도시의 에너지 설계를 대신하는 전문 공학 계산 도구가 아닙니다.  
> 프로그램의 기본 공급량, 효율, 점수와 비교 결과는 학생의 탐구와 토론을 위한 **수업용 가정값**입니다.

## 1. 프로그램 소개

`E-CITY 2050 미래학교 에너지 시스템`은 학생이 단순히 친환경 에너지를 많이 선택하는 활동에서 벗어나, 다음 질문에 데이터와 근거를 이용해 답하도록 돕는 수업용 프로그램입니다.

- 전기는 어디에서 만들어져 학교까지 오는가?
- 모든 발전 시설을 학교 안에 설치할 수 있는가?
- 발전, 저장, 절감, 관리와 외부 공급은 어떤 차이가 있는가?
- 학교는 하루 중 언제 전기를 가장 많이 사용하는가?
- 태양광이 생산되지 않는 밤에는 어떤 방식으로 전기를 공급해야 하는가?
- ESS는 발전 시설인가, 저장 시설인가?
- 외부 전력망을 사용하는 설계는 실패한 설계인가?
- 수소 연료전지는 언제나 친환경이라고 할 수 있는가?
- 두 설계안 중 어느 것이 더 안정적이고 현실적인가?
- 실제 적용 전 어떤 자료를 추가로 조사해야 하는가?

학생은 발전 기술의 원리와 장소 조건을 살펴보고, 시간별 전력 사용 데이터를 읽은 뒤, 태양광·ESS·수소 연료전지·에너지 절감·스마트 관리를 조절합니다. 이후 설계안 A와 B를 비교하고 최종안을 선택하여 모둠 보고서와 발표문을 작성합니다.

## 2. 개발 취지

### 2.1 에너지원 나열에서 에너지 시스템 사고로

에너지 수업은 태양광, 풍력, 원자력 등의 장단점을 각각 외우는 활동으로 끝나기 쉽습니다. 그러나 실제 학교의 에너지 문제는 하나의 기술만 선택하여 해결할 수 없습니다.

이 프로그램은 다음 요소를 하나의 시스템으로 연결하도록 설계되었습니다.

- 전기를 만드는 **발전**
- 남는 전기를 보관하는 **저장**
- 필요한 전력 자체를 줄이는 **절감**
- 전력의 흐름과 피크를 조절하는 **관리**
- 부족한 전기를 공급하는 **외부 전력망**

학생은 특정 기술이 “좋다” 또는 “나쁘다”라고 단정하는 대신, 기술의 역할과 적용 장소, 시간대별 수요, 안전·공간·공급 조건을 함께 판단합니다.

### 2.2 실제 데이터를 읽는 힘 기르기

학생은 수업용 24시간 데이터 또는 교사가 준비한 CSV를 이용하여 다음 내용을 확인합니다.

- 총 전력 사용량
- 시간당 평균 사용량
- 피크 시간과 피크 사용량
- 최저 사용 시간
- 시간별 전력 사용 패턴
- 태양광 발전 가능 시간과 피크 시간의 관계

그래프에 나타난 시간 패턴만으로 원인을 확정하지 않고, 냉난방·급식·조명·컴퓨터·방과후 활동 등 추가 자료가 필요할 수 있음을 인식하도록 합니다.

### 2.3 장소 적합성을 포함한 기술 선택

프로그램은 발전 기술을 학교 안에 무조건 배치하지 않습니다.

학생은 기술을 다음 세 영역으로 분류합니다.

| 분류 | 의미 | 대표 예 |
|---|---|---|
| 학교 안 | 학교 건물이나 부지에 적용 가능 | 태양광, ESS, 에너지 절감, 에너지관리시스템 |
| 조건부 지역 | 공급·안전·공간·자연조건 확인 후 지역에서 검토 | 풍력, 수소 연료전지 |
| 외부 전력망 | 학교 밖 발전소에서 생산하여 전력망으로 공급 | 화력, 원자력, 수력 |

원자력은 학교 안 설계 슬라이더에 포함하지 않으며, 학교 밖 발전소에서 생산되어 외부 전력망을 통해 공급되는 전원의 한 종류로 다룹니다.

### 2.4 정답 경쟁보다 근거 있는 의사결정

지역 에너지 충당률이 높다고 무조건 좋은 설계로 평가하지 않습니다.

학생은 다음 내용을 함께 비교해야 합니다.

- 지역 에너지 충당률
- 외부 전력망 사용량
- 피크 시간 대응
- 밤 시간 외부망 의존
- 태양광 잉여 전력
- ESS 충전·방전
- 기술의 장소 적합성
- 안전·비용·환경성의 한계
- 실제 적용 전 추가 조사 항목

## 4. 핵심 개념 구분

### 4.1 발전

에너지를 전기에너지로 전환합니다.

- 화력 발전
- 원자력 발전
- 수력 발전
- 풍력 발전
- 태양광 발전
- 수소 연료전지

### 4.2 저장

이미 생산된 전기를 저장했다가 필요할 때 다시 공급합니다.

- ESS

> ESS 방전량은 새로 만든 발전량이 아닙니다.

### 4.3 절감

학교가 필요로 하는 전력 사용량 자체를 줄입니다.

- LED 조명
- 단열 개선
- 고효율 냉난방
- 대기전력 차단
- 사용 습관 개선

### 4.4 관리

전기를 만들지는 않지만, 사용 시간과 저장 장치의 작동을 조절합니다.

- 스마트 에너지관리시스템
- 센서와 전력계
- 피크 수요 관리
- ESS 충전·방전 관리

### 4.5 공급과 연결

학교 내부 설비로 부족한 전기를 외부에서 안정적으로 공급합니다.

- 외부 전력망
- 마이크로그리드 개념

> 외부 전력망을 사용하는 것은 실패가 아닙니다. 실제 학교도 전력망과 연결되어 필요한 전기를 공급받습니다.

## 5. 프로그램의 다섯 가지 미션

### 미션 1. 전기는 어디서?

학생은 발전소에서 학교까지 전기가 전달되는 과정을 확인합니다.

```text
발전소 → 송전선 → 변전소 → 지역 전력망 → 학교
```

주요 개념:

- 화력과 원자력은 열을 얻는 방식은 다르지만 증기·터빈·발전기를 이용합니다.
- 수력과 풍력은 물과 바람의 운동으로 터빈을 돌립니다.
- 태양광은 터빈 없이 빛에너지를 전기에너지로 직접 전환합니다.
- 연료를 태우거나 핵분열이 일어난다고 전기가 바로 생기는 것은 아닙니다.

### 미션 2. 학교 주변 적합성

학생은 기술 카드를 읽고 학교 안·조건부 지역·외부 전력망으로 분류합니다.

필수 분류 기술:

- 태양광
- 풍력
- 수소 연료전지
- 원자력

학생은 단순히 위험 여부만 판단하지 않고 다음 조건을 함께 살펴야 합니다.

- 필요한 공간
- 자연조건
- 안전시설
- 연료와 자원의 공급
- 소음과 주변 환경
- 전문 관리
- 비용
- 법과 지역사회 수용성

### 미션 3. 전력 패턴 탐정

학생은 수업용 데이터 또는 CSV를 불러와 전력 사용 패턴을 분석합니다.

확인하는 값:

- 총 사용량
- 평균 사용량
- 피크 시간
- 피크 사용량
- 최저 시간
- 시간대별 그래프

피크 원인을 학생이 직접 추론하고, 그래프만으로 원인을 확정할 수 있는지 검토합니다.

### 미션 4. 미래학교 시스템

학생은 다섯 개의 조절 항목을 사용하여 에너지 시스템을 설계합니다.

- 학교 지붕·주차장 태양광
- ESS 용량
- 조건부 수소 연료전지
- 에너지 절감률
- 스마트 에너지 관리

현재 설정을 설계안 A와 B에 각각 저장하고, 두 설계안을 비교한 뒤 최종안을 선택합니다.

### 미션 5. 보고서와 발표

학생은 다음 내용을 포함한 보고서를 작성합니다.

- 모둠 이름
- 미래학교 또는 학교구역 이름
- 데이터 출처
- 피크 시간과 원인 추론
- 학교 안에 적용한 기술
- 조건부로 적용한 기술
- 외부 전력망을 통해 이용한 기술
- 지역 에너지 충당률
- 외부 전력망 사용량
- 설계의 장점
- 설계의 한계
- 실제 적용 전 추가 조사
- 1분 발표문

## 6. 학생 사용 방법

### 6.1 프로그램 시작

1. 교사가 제공한 학생용 주소에 접속합니다.
2. 기본 URL에 `?mode=teacher`가 붙어 있지 않은지 확인합니다.
3. 첫 화면에서 활동의 핵심 질문과 주의사항을 읽습니다.
4. `미래학교 에너지 미션 시작하기`를 선택합니다.

### 6.2 미션 진행

프로그램은 이전 미션을 완료해야 다음 미션을 사용할 수 있도록 구성되어 있습니다.

1. 전력 전달 과정에 관한 개념 문항을 해결합니다.
2. 기술 카드를 읽고 장소 적합성을 분류합니다.
3. 개념 확인 문항을 해결합니다.
4. 수업용 데이터 또는 교사가 제공한 CSV를 불러옵니다.
5. 피크 시간과 피크 원인을 확인합니다.
6. 설계안 A와 B를 만들어 저장합니다.
7. 두 설계 중 하나를 최종안으로 선택합니다.
8. 보고서 초안을 확인하고 모둠의 생각에 맞게 수정합니다.
9. 결과를 복사하거나 인쇄합니다.

### 6.3 진행 내용 자동 저장

학생의 진행 내용은 현재 브라우저 탭의 `sessionStorage`에 자동 저장됩니다.

- 같은 탭에서 새로고침하면 이어서 진행할 수 있습니다.
- 탭을 닫으면 저장 상태가 사라질 수 있습니다.
- 공용기기에서는 수업 후 `처음부터`를 실행한 뒤 탭을 닫습니다.
- 학생 이름과 학번 대신 모둠 별칭을 사용합니다.

## 7. 교사 사용 방법

### 7.1 학생 모드와 교사 모드

| 모드 | 접속 방법 | 용도 |
|---|---|---|
| 학생 모드 | 기본 URL | 학생 활동 |
| 교사 모드 | URL 끝에 `?mode=teacher` | 수업용 가정값 조절, CSV 매핑, 출처 상세 입력, JSON 저장 |

예시:

```text
학생: https://배포주소/
교사: https://배포주소/?mode=teacher
```

> `?mode=teacher`는 인증 기능이 아닙니다.  
> 교사 모드에 평가 정답, 개인정보, API 키와 같은 민감정보를 넣어서는 안 됩니다.

학생 모드는 브라우저에 저장된 교사용 가정값을 사용하지 않고 기본 수업 가정값으로 시작합니다.

### 7.2 수업 데이터 준비

교사는 다음 방법 중 하나를 선택할 수 있습니다.

#### 방법 A. 수업용 가정 데이터

빠른 수업 진행이나 기기 오류 대안으로 사용합니다.

- 24시간 데이터
- 하루 총 사용량 900 kWh
- 피크 시간 14시
- 피크 사용량 67 kWh

이 데이터는 실제 학교나 지역의 전력 사용량이 아닙니다. 화면과 보고서에는 `수업용 가정 데이터`로 표시됩니다.

#### 방법 B. 교사가 준비한 CSV

학교 또는 공공데이터를 정리해 업로드합니다.

권장 형식:

```csv
date,hour,region,usage_kWh
2026-07-08,14,개포동,305
```

한국어 열 이름도 사용할 수 있습니다.

```csv
날짜,시간,지역,전력사용량
2026-07-08,14,개포동,305
```

앱이 열 이름을 찾지 못하면 교사 모드에서 날짜·시간·지역·전력사용량 열을 직접 지정할 수 있습니다.

### 7.3 공공데이터 출처 입력

교사가 준비한 공공데이터를 사용하는 경우 다음 정보를 기록합니다.

- 제공기관
- 자료명
- 기준일
- 지역 단위
- 학교 자체 데이터인지 여부
- 지역 대체 데이터인지 여부

파일명만으로 공공데이터라고 판정하지 않습니다.

개포동 시간별 데이터를 사용할 경우 학교 자체 전력량이 아니라 지역 단위의 대체 자료일 수 있으므로 보고서에도 `지역 대체 데이터`라고 표시해야 합니다.

### 7.4 교사용 가정값 조절

교사 모드에서는 다음 값을 바꿀 수 있습니다.

- 태양광 100% 설정 시 활동 시간당 공급량
- 수소 연료전지 100% 설정 시 시간당 공급량
- ESS 최대 저장 용량
- ESS 시간당 최대 충전량
- ESS 시간당 최대 방전량
- ESS 왕복 효율
- 절감 최대 비율
- 스마트 관리의 피크 추가 절감 상한
- 전력망 배출계수
- 태양광 발전 시작·종료 시간
- 수소 생산 방식

수소 생산 방식:

| 설정 | 프로그램 해석 |
|---|---|
| `green` | 재생전력 기반 생산을 가정하여 환경성 비교 점수에 제한적으로 반영 |
| `mixed` | 환경성 점수 가산 없음 |
| `unspecified` | 생산 방식 미확인, 환경성 점수 가산 없음 |

모든 값은 공식 설계값이 아니라 수업용 비교 가정값입니다.

## 8. CSV 처리 방식

### 지원 사항

- 최대 파일 크기: 1MB
- UTF-8
- UTF-8 BOM
- CP949/EUC-KR
- 영어 및 한국어 열 이름 자동 감지
- 교사 모드 수동 컬럼 매핑
- 쉼표가 포함된 숫자 처리
- `14`, `14시`, `14:00` 형태의 시간 처리
- 유효하지 않은 행 제외
- 제외 행 수 안내
- 잘못된 새 파일을 선택해도 기존 정상 데이터 유지

### 필수 데이터

| 열 | 필수 여부 | 설명 |
|---|---|---|
| 날짜 | 선택 | 여러 날짜를 구분할 때 사용 |
| 시간 | 필수 | 0~23 |
| 지역 | 선택 | 학교·지역명 |
| 전력사용량 | 필수 | 0 이상의 kWh 값 |

### 여러 날짜가 있는 데이터

- 날짜가 한 개인 경우 시간별 값은 합계 형태로 표시합니다.
- 날짜가 두 개 이상인 경우 같은 시간대의 평균 사용량으로 그래프를 구성합니다.
- 전체 총 사용량은 모든 유효 행의 합계입니다.

## 9. 시간별 에너지 모델

### 9.1 계산 순서

CSV의 각 행을 시간 순서로 처리하며 ESS의 초기 저장량은 0 kWh입니다.

1. 기본 에너지 절감률을 적용합니다.
2. 원 데이터의 피크 시간에는 스마트 관리의 추가 절감을 적용합니다.
3. 설정된 낮 시간에 태양광 발전량을 계산합니다.
4. 조건부 수소 연료전지 공급량을 계산합니다.
5. 태양광을 현재 수요에 먼저 사용합니다.
6. 수소 연료전지가 남은 수요를 보충합니다.
7. 남은 태양광을 ESS에 충전합니다.
8. 부족한 전력을 ESS에서 방전합니다.
9. 그래도 부족한 전력을 외부 전력망에서 공급합니다.
10. 사용하거나 저장하지 못한 전력을 잉여 전력으로 기록합니다.

### 9.2 태양광

```text
시간당 태양광 발전량 =
교사용 최대 공급 가정값 × 학생 태양광 슬라이더 비율
```

태양광은 교사가 설정한 활동 시간에만 발전합니다. 실제 일사량의 시간별 곡선, 날씨, 계절, 지붕 방향과 음영은 현재 모델에 포함되지 않습니다.

### 9.3 ESS

ESS 비율은 다음 값을 함께 조절합니다.

- 최대 저장 용량
- 시간당 충전 한도
- 시간당 방전 한도

ESS는 태양광 잉여 전력만 충전합니다. 충전 시 왕복 효율 손실을 반영하며, 저장 용량과 시간당 충전·방전 한계를 넘지 않습니다.

> ESS 방전량은 저장된 전력의 이동이며 새로운 발전량이 아닙니다.

### 9.4 스마트 관리

스마트 관리 슬라이더는 발전량에 더해지지 않습니다.

현재 수업용 모델에서는 원 데이터의 피크 시간대 수요를 교사가 정한 최대 범위 안에서 추가로 줄입니다.

### 9.5 지역 에너지 충당률

```text
지역 에너지 충당률 =
(절감 후 소비량 - 외부 전력망 사용량)
÷ 절감 후 소비량 × 100
```

- 0~100%로 표시합니다.
- 외부 전력망 공급량은 지역 충당률에 포함하지 않습니다.
- 학교 밖 원자력 발전을 지역 생산량으로 포함하지 않습니다.
- 지역 충당률이 높다고 무조건 좋은 설계는 아닙니다.

## 11. 설계안 A/B 비교 방법

1. 태양광·ESS·수소·절감·스마트 관리 값을 조절합니다.
2. 현재 결과를 확인합니다.
3. `A에 저장`을 선택합니다.
4. 다른 방식으로 값을 조절합니다.
5. `B에 저장`을 선택합니다.
6. A와 B를 각각 불러와 결과를 비교합니다.
7. 다음 질문을 토론합니다.

- 어느 설계가 외부 전력망 사용량이 적은가?
- 어느 설계가 밤 시간에 더 안정적인가?
- ESS 용량을 늘렸는데도 효과가 작다면 왜 그런가?
- 태양광이 너무 많아 잉여 전력이 생기는가?
- 수소 연료전지를 사용할 장소와 공급 조건은 확인되었는가?
- 절감과 스마트 관리가 수업 환경을 지나치게 제한하지 않는가?
- 실제 적용 가능성은 어느 설계가 높은가?

8. 최종 설계안을 선택합니다.

## 12. 보고서와 발표문

프로그램은 현재 데이터와 최종 설계를 바탕으로 보고서 초안을 만듭니다.

자동 초안에는 다음 내용이 포함됩니다.

- 데이터 출처 상태
- 피크 시간
- 피크 원인 추론
- 학교 안·조건부 지역·외부 공급 기술
- 지역 에너지 충당률
- 외부 전력망 사용량
- 태양광·수소·ESS 상세 흐름
- 설계의 장점
- 설계의 한계
- 추가 조사 항목
- 발표문 초안

학생은 자동 초안을 그대로 제출하지 않고 모둠의 판단에 맞게 수정해야 합니다.

수업용 가정 데이터를 사용하면 보고서 상단에 다음 경고가 자동으로 들어갑니다.

> 이 보고서는 실제 학교 전력량이 아닌 수업용 가정 데이터로 작성되었습니다.

## 13. 권장 수업 운영

### 권장 환경

- 2차시
- 차시당 45~50분
- 35명 학급
- 4명 안팎의 모둠
- 모둠별 스마트폰·태블릿·노트북 1대 이상
- 학생 실명 대신 모둠 별칭 사용

### 1차시: 발전 원리와 장소 적합성

| 시간 | 학생 활동 | 교사의 지원 |
|---|---|---|
| 0~5분 | 핵심 질문 읽기 | 학교 콘센트 뒤에 연결된 시설을 질문 |
| 5~15분 | 발전소에서 학교까지 전력 여정 확인 | 화력·원자력의 공통점과 태양광의 차이 설명 |
| 15~30분 | 에너지 기술 카드 탐색 | 공간·안전·자연조건·공급·비용을 함께 보게 함 |
| 30~40분 | 장소 적합성 분류와 이유 작성 | 원자력·수소·ESS에 관한 대표 오개념 점검 |
| 40~50분 | 개념 확인과 설명 수정 | 처음 생각과 달라진 점 발표 |

### 2차시: 데이터 분석과 시스템 설계

| 시간 | 학생 활동 | 교사의 지원 |
|---|---|---|
| 0~8분 | 데이터 출처 확인 및 불러오기 | 실제·수업용·지역 대체 자료 구분 |
| 8~18분 | 피크·최저 시간과 원인 추론 | 그래프만으로 원인을 확정하지 않도록 지도 |
| 18~33분 | 에너지 시스템 조절, A/B 저장 | ESS·스마트 관리가 발전원이 아님을 확인 |
| 33~40분 | 두 설계 비교 및 최종 선택 | 충당률 경쟁이 되지 않도록 한계와 현실성 질문 |
| 40~50분 | 보고서 수정과 1분 발표 | 장소 구분 이유와 추가 조사를 포함하게 함 |

### 속도 차이 대응

- 빠른 모둠: 수소 생산 방식을 바꾸어 환경성 결과 비교
- 보통 모둠: 수업용 데이터로 A/B 설계와 보고서 완성
- 도움이 필요한 모둠: 태양광·ESS·절감만 조절하고 외부망 부족분 해석

## 14. 기기와 네트워크 문제 대응

- CSV가 열리지 않으면 `수업용 데이터로 시작하기`를 사용합니다.
- 앱이 이미 열려 있다면 핵심 활동은 외부 API 없이 진행할 수 있습니다.
- 앱 접속이 되지 않으면 24시간 전력표와 기술 분류표를 인쇄하여 같은 순서로 진행합니다.
- 한 모둠의 기기가 멈추면 다른 모둠 화면을 참고해 종이 활동지에 A/B 값을 기록합니다.
- 브라우저 저장이 차단되면 보고서를 복사하거나 인쇄합니다.
- 교사 모드에서는 상태를 JSON 파일로 별도 저장할 수 있습니다.

- 로그인 데이터베이스가 없습니다.
- 분석 SDK가 없습니다.

- 외부 웹폰트를 사용하지 않습니다.

- 공용기기에서는 이전 모둠 기록을 초기화합니다.
- `?mode=teacher`를 보안 기능으로 사용하지 않습니다.
- 앱 결과만으로 고부담 평가나 중앙 성적 산출을 하지 않습니다.
- 실제 공개 배포 후 HTTPS와 보안 응답 헤더를 별도로 확인합니다.

## 16. 현재 구현 상태

### 구현된 기능

- 5단계 미션형 수업 흐름
- 이전 미션 완료 전 다음 활동 잠금
- 발전 원리와 전력 전달 과정 학습
- 기술별 원리·조건·장점·한계·오개념 안내
- 학교 안·조건부 지역·외부 전력망 분류
- 개념 확인 문항
- 수업용 24시간 데이터
- CSV 업로드와 자동 열 감지
- 한국어 열 이름과 수동 컬럼 매핑
- UTF-8·BOM·CP949/EUC-KR 읽기
- 시간별 사용량 그래프
- 피크 시간과 피크 원인 기록
- 시간별 태양광·수소·ESS·외부망 계산
- 태양광·ESS·수소·절감·스마트 관리 슬라이더
- 설계안 A/B 저장·불러오기·최종 선택
- 지역 충당률과 상세 에너지 흐름
- 보고서 자동 초안·수정·복사·인쇄
- 현재 탭 자동 저장
- 교사 모드 브라우저 저장과 JSON 저장
- 전체 수업 초기화
- Vitest 단위·컴포넌트 테스트
- GitHub Actions 테스트와 프로덕션 빌드

### 현재 제한

1. **실제 공학 설계 도구가 아닙니다**  
   설치 면적, 계절별 일사량, 기상, 실제 설비 효율과 비용을 정밀하게 계산하지 않습니다.

2. **태양광 발전 곡선을 단순화합니다**  
   설정된 활동 시간 동안 같은 시간당 발전량을 사용합니다.

3. **스마트 관리를 단순화합니다**  
   원 데이터의 피크 시간에만 추가 절감 효과를 적용합니다.

4. **ESS 모델이 교육용으로 단순화되어 있습니다**  
   배터리 종류, 열관리, 수명 저하, 충·방전 전략과 비상운전은 포함하지 않습니다.

5. **수소의 전 과정 환경성을 계산하지 않습니다**  
   생산 원료·전력, 압축, 저장, 운송과 누출 영향을 직접 계산하지 않습니다.

6. **비용 모델이 없습니다**  
   초기 설치비, 운영비, 유지보수비, 교체비와 회수 기간을 계산하지 않습니다.

7. **실시간 날씨와 공공데이터 API를 연결하지 않습니다**  
   교사가 CSV를 직접 준비해야 합니다.

8. **학생 결과를 중앙 서버에 모으지 않습니다**  
   개인정보 보호에는 유리하지만 교사가 모둠 결과를 자동 수집하는 기능은 없습니다.

9. **교사 모드는 인증되지 않습니다**  
   URL 쿼리로 열리는 로컬 설정 화면입니다.

10. **실제 교실 적용 검증이 필요합니다**  
    학생 이해도, 소요 시간, 기기별 사용성, 오개념 변화에 대한 현장 검증이 필요합니다.

## 18. 교사용 수업 전 체크리스트

- [ ] 학생용 URL에 `?mode=teacher`가 없다.
- [ ] 수업용 가정 데이터가 실제 학교 데이터가 아님을 안내했다.
- [ ] 실제 CSV의 제공기관과 자료명을 확인했다.
- [ ] 학교 자체 데이터와 지역 대체 데이터를 구분했다.
- [ ] CSV의 시간 단위와 전력량 단위를 확인했다.
- [ ] 학생 개인정보가 CSV에 포함되지 않았다.
- [ ] 원자력을 학교 안 시설로 설명하지 않는다.
- [ ] ESS를 발전원으로 설명하지 않는다.
- [ ] 스마트 관리를 발전량으로 더하지 않는다.
- [ ] 수소 생산 방식이 불명확할 때 친환경성을 단정하지 않는다.
- [ ] 외부 전력망 사용을 설계 실패로 평가하지 않는다.
- [ ] 지역 충당률 하나만으로 설계 순위를 정하지 않는다.
- [ ] 보고서에 장소 적합성의 이유가 포함된다.
- [ ] 보고서에 실제 적용 전 추가 조사 항목이 포함된다.
- [ ] 공용기기 초기화 절차를 안내했다.
- [ ] 기기·네트워크 오류 시 사용할 인쇄 자료를 준비했다.

## 19. 문제 해결

| 문제 | 확인할 내용 |
|---|---|
| 다음 미션으로 이동할 수 없음 | 이전 미션의 필수 문항·기술 카드·분류·피크 확인이 완료되었는지 확인합니다. |
| CSV 파일이 열리지 않음 | 파일이 1MB 이하인지, `.csv` 형식인지 확인합니다. |
| 한글 CSV가 깨짐 | UTF-8 또는 CP949/EUC-KR로 저장했는지 확인합니다. 앱이 인코딩 상태를 안내합니다. |
| 시간 열을 찾지 못함 | 교사 모드에서 시간 열을 직접 지정합니다. |
| 전력사용량 열을 찾지 못함 | 교사 모드에서 전력사용량 열을 직접 지정합니다. |
| 일부 행이 사라짐 | 시간 범위가 0~23인지, 사용량이 숫자이고 음수가 아닌지 확인합니다. |
| 총 사용량과 그래프 값이 다르게 보임 | 여러 날짜 자료에서는 그래프가 같은 시간대의 평균을 보여 주지만 총 사용량은 전체 행의 합계입니다. |
| 태양광을 높여도 밤 사용량이 줄지 않음 | 태양광은 설정된 낮 시간에만 발전합니다. ESS에 충전된 전력이 있어야 밤에 사용할 수 있습니다. |
| ESS를 높여도 효과가 없음 | 낮 시간 잉여 태양광이 없으면 ESS에 충전할 전력이 없습니다. |
| 수소 환경성 점수가 오르지 않음 | 교사 모드의 수소 생산 방식이 `green`인지 확인합니다. |
| 외부망 사용량이 남음 | 현재 지역 설비와 저장량으로 부족한 전력을 정상적으로 외부망에서 공급한 것입니다. |
| 보고서가 이전 값으로 남음 | 자동 초안 다시 만들기를 실행합니다. |
| 새로고침 후 진행이 사라짐 | 같은 탭인지 확인하고, 브라우저의 sessionStorage 사용이 차단되지 않았는지 확인합니다. |
| 이전 모둠 기록이 보임 | `처음부터`를 실행해 브라우저 저장 내용을 지웁니다. |

## 25. 프로그램의 지향점

이 프로그램의 목적은 학생이 태양광 비율을 최대한 높여 높은 점수를 얻는 것이 아닙니다.

학생이 다음과 같이 설명할 수 있는 수업을 지향합니다.

> “학교의 전력 사용은 시간에 따라 달라지며, 태양광은 낮에만 발전한다.  
> ESS는 남는 전기를 저장하고, 절감과 스마트 관리는 필요한 전력과 피크를 줄인다.  
> 학교 안 설비로 부족한 전기는 외부 전력망에서 공급받을 수 있다.  
> 따라서 미래학교의 에너지 시스템은 하나의 발전원이 아니라 여러 기술과 전력망을 연결하여 설계해야 한다.”

데이터의 출처와 한계를 확인하고, 기술의 역할과 장소 적합성을 구분하며, 실제 적용 전에 필요한 추가 조사를 스스로 제안하는 **근거 기반 K-STEAM 에너지 시스템 설계 수업 플랫폼**으로 발전시키는 것이 최종 방향입니다.
